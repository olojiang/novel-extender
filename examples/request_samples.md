# 请求样例

这些样例可以作为 `build-context` 的第二个参数使用。

## 分析

```text
分析主角、北楼、旧徽章三条线索之间的关系，列出已出现伏笔、尚未解释的信息，以及后续三章可以推进的方向。
```

```text
整理目前所有关键人物的目标、误解、秘密和冲突点，指出哪些人物关系最适合在下一章升级。
```

## 续写

```text
续写下一章：主角夜探北楼，发现旧徽章和档案室失踪名单有关。保持悬疑节奏，不要提前揭示幕后人。
```

```text
续写下一章：让配角误以为主角背叛团队，制造一次公开冲突，同时埋下旧照片的新线索。
```

## 改写章节

```text
改写第2章：压缩开头铺垫，增强主角发现旧徽章时的紧张感，保留档案线索和人物关系。
```

```text
重写第5章的后半段：把说明性叙述改成对话和行动，不改变结论，不提前揭示反派身份。
```

## 连贯性检查

```text
检查最近三章与旧章节之间的人物动机、时间线、地点移动和道具线索是否矛盾，并给出修正建议。
```
